Error in defining arrays in C++ (Variable Sized Object)

Ravi Kumar (Junior - 1st Year) | Posted today at 10:00 AM | Subject: Programming in C++

Hi seniors, I'm trying to declare an array in C++ like `int n; cin >> n; int arr[n];`. My code runs sometimes but sometimes gives a segmentation fault. My professor said variable sized object may not be intialized. What should I use instead?

Megha Sharma (Senior - 3rd Year)

Hey Ravi! Standard C++ does not support Variable Length Arrays (VLAs). Sometimes GCC compilers allow it as an extension, which is why it works sometimes. You should dynamically allocate memory using `int* arr = new int[n];` or better yet, use the Standard Template Library (STL) vectors: `#include ` and then `std::vector arr(n);`

Ravi Kumar (Junior - 1st Year)

Got it! I will start studying vectors. Thank you so much!

Which Database is better for a Real-Time Chat App?

Karan Singh (2nd Year) | Posted 2 days ago | Web Development

I am planning to build a real-time student chat application for my semester project using Node.js. Should I go with MongoDB or Firebase for the database? Any insights from seniors who built similar projects?

Arjun Reddy (Senior - 4th Year)

Firebase Realtime Database or Firestore makes building it much faster since WebSockets and real-time syncing are handled out of the box. But if your goal is to strictly learn backend architecture and Socket.io, use Node.js + MongoDB + Socket.io.
